# Интерактивный Диалог с AI

## 🎯 Что это

Режим живого общения с AI для проектирования схем. Похоже на Claude Code или Qwen Code.

## 🚀 Использование

### Быстрый старт

```bash
cd /Users/vladglazunov/Documents/algo/PCBAI
source venv/bin/activate

# Запустить диалог
pcba dialog
```

### Пример диалога

```
============================================================
  PCBAI Interactive Schematic Designer
============================================================

👋 Привет! Я AI-помощник для проектирования схем.
Опиши какую схему хочешь создать, например:
  • 'Светодиод с резистором к 5V'
  • 'ATmega328P с DHT22 и BMP280'
  • 'ESP32 с OLED дисплеем'

Команды:
  • 'exit' / 'quit' / 'выход' - выйти
  • 'save <filename>' - сохранить схему
  • 'show' - показать текущую схему
  • 'help' - помощь
============================================================

🔹 Вы: Хочу схему с датчиком температуры

🤖 AI: Отлично! Для измерения температуры рекомендую:
  • DHT22 - цифровой датчик температуры и влажности
  • BMP280 - датчик температуры и давления (I2C)
  
Какой интерфейс предпочитаете?
  1) I2C (2 провода + питание)
  2) 1-wire (1 провод + питание)

Также понадобятся:
  • Подтягивающий резистор 4.7k для I2C или 10k для 1-wire
  • Развязывающий конденсатор 100nF

🔹 Вы: Давайте DHT22 с Arduino Uno

🤖 AI: Хороший выбор! Схема будет такой:

Компоненты:
  • U1: ATmega328P (Arduino Uno)
  • D1: DHT22 (датчик температуры/влажности)
  • R1: 10k резистор (pull-up для DATA)
  • C1: 100nF конденсатор (развязка питания)

Подключение:
  • DHT22 VCC → 5V
  • DHT22 GND → GND
  • DHT22 DATA → Pin 2 Arduino + 10k к 5V

Сохранить схему? (save filename.kicad_sch)

🔹 Вы: save temperature_sensor.kicad_sch

✅ Схема сохранена: temperature_sensor.kicad_sch
✅ KiCad validation: OK
```

## 📝 Команды диалога

| Команда | Описание |
|---------|----------|
| `help` | Показать справку |
| `show` | Показать текущую схему |
| `save <filename>` | Сохранить схему в файл |
| `exit` / `quit` / `выход` | Выйти |

## 🔧 Настройка

### required: LLM API

Перед использованием настройте API (см. `API_SETUP.md`):

**Вариант 1: Google AI Studio (рекомендуется)**
```bash
# Получить ключ: https://aistudio.google.com/app/apikey
# Добавить в knowledge_base/config.json:
{
  "default_provider": "google",
  "llm_providers": {
    "google": {"enabled": true, "api_key": "YOUR_KEY"}
  }
}
```

**Вариант 2: Puter.js (без ключа)**
```json
{
  "default_provider": "puter"
}
```

**Вариант 3: Ollama (локально)**
```bash
brew install ollama
ollama serve &
ollama pull llama3.2
```

## ✅ Валидация

После сохранения схема автоматически проверяется через `kicad-cli` (если установлен).

**Установить KiCad:**
- macOS: `brew install --cask kicad`
- Linux: `sudo apt install kicad`
- Windows: https://www.kicad.org/download/

**Проверка вручную:**
```bash
pcba validate temperature_sensor.kicad_sch
```

## 🐛 Troubleshooting

### Ошибка: "All LLM providers failed"
**Решение:** Настроить API (см. `API_SETUP.md`)

### Ошибка: "Could not parse JSON"
**Решение:** Попробовать другой провайдер или упростить запрос

### Ошибка валидации KiCad
**Решение:** Проверить формат файла вручную или через `pcba validate`
